サービスを作った理由

　

現在、出会った事のない人同士を繋ぐサービスは、多々ありますが既に、出会った事のある人同士を再び繋ぐサービスが、facebookしか見つかりませんでした。
しかしFacebookは、多数と繋がる事が前提なので

  • 多数とは繋がりがたくないので登録していない(自分はこのケース)
  • 検索制限をかけている
  • 苗字が変わっている

    • などのケースでは見つけられません。そこで繋がりたい人を指定することで
  • 多数に見られないので安心
  • 繋がりたい人からは、検索される事が出来る
  • 繋がりたい人が知っている名前で登録すれば名前でのミスマッチが起きない。

    • 等により、今でも気になる過去の人、Facebookで見つからなかった人、を再び繋げる事が出来ると考えてサービスを作りました

     

     

    Flow

    実装機能

    ユーザー新規登録、ログイン（ユーザーメールアドレス、googoleアカウント)、パスワード忘れによる再設定、ログアウト、退会、投稿リスト、ユーザー情報の投稿、編集、マッチユーザーの検索、ユーザーブロック機能、お問い合わせ

    使用技術

    mBaaS Firebase

  • RealtimeDataBase（投稿データ）
  • Firebase Authentication（ユーザー管理,ユーザー毎にID付与）
  • FireStorage（画像）

    • データの安全性を高める目的と、現場に近い環境で開発がしたかったので開発環境とリリース用でFirebaseのプロジェクト（データの保存先）をBuildのTarget毎に処理を切り分けました

    Code管理 Github

    ソースコードのバージョン管理はGithubで行いました。開発は１人ですがメンター様とのコードレビューにも使用する事で現場でのコード管理を疑似的に練習しました。

    GUI SourceTree

    ターミナルで基本操作を身につけた後にGitクライアントSourceTreeを導入しGit管理の工数を減らす事が出来ました。

    License管理 LicensePlist

    OSSライセンスの表記は初心者であっても遵守しなければなりません。LicensePlistこれ１つで全て賄ってくれるのでライセンス管理工数を大幅に削減出来ました。

    Logger XCGLogger

    ロギングライブラリ を導入し、デバッグスピードを上げる事が出来ました。

    広告 Google AdMob

    TabBar上部にタブ形式で広告を表示させました。

    UX/UI XLPagerTabStrip

    SignUp/LogInをスワイプで切り替えにしました。NewsPicksのログインUIを参考に、XLPagerTabStripで同じUIを実装出来ました。自力で既存のappと同じUIを実装しました

    写真加工 RSKImageCropper

    写真ライブラリを立ち上げ選択した写真を丸く切り抜きます。
    写真投稿時の、フォトライブラリ立ち上げ→写真選択→切り抜き範囲選択→画像位置調整等、工数のかかる処理をRSKImageCropperの導入で短時間で実装しました。今後はライブラリ無しでも、実装できるスキルを身に付けたいと思います。
